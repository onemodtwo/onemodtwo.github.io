SETTING UP PYSPARK TO WORK IN CONDA ENVIRONMENTS

The following recipe will allow you to install a version of PySpark on your machine and then use it in any conda environment, either in an interactive python command line session, an interactive ipython session or in a jupyter console session or notebook.

Steps 1 â€“ 3 below only need to be done once when first installing a version of PySpark. Steps 4 and 5 are required for each conda environment wishing to use this version of PySpark.

  1. Make sure Spark is downloaded and expanded into ~/bin/spark/sparkX.Y.Z, where X.Y.Z is the version number.

  2. Make sure that there is a corresponding profile in ~/.ipython named profile_pysparkX.Y.Z, where X, Y, and Z are as above. If newly installing this version of Spark, this profile can be created by issuing the command

    $ ipython profile create pysparkX.Y.Z

    This will create a default profile in ~/.ipython/profile_pysparkX.Y.Z. Inside this directory is one called startup. Create a file inside there called 00-pysparkX.Y.Z-setup.py. The contents of this file should be:

    import os
import sys
import fnmatch

spark_home = os.path.expanduser('~/bin/spark/sparkX.Y.Z')

for file in os.listdir(os.path.join(spark_home, 'python/lib')):
    if fnmatch.fnmatch(file, 'py4j-*-src.zip'):
        py4j_version = 'py4j-' + file.split('-')[1] + '-src.zip'

sys.path.insert(0, os.path.join(spark_home, 'python'))
sys.path.insert(0, os.path.join(spark_home, 'python/lib/', py4j_version))
exec(open(os.path.join(spark_home, 'python/pyspark/shell.py')).read())

    NOTE: Change X.Y.Z to the Spark version number.

  3. Inside the root conda directory (e.g. ~/anaconda, ~/miniconda, etc.) create a directory called kernels/pysparkX.Y.Z; i.e., <conda-root-dir>/kernels/pysparkX.Y.Z. (The kernels subdirectory may already exist if there are already pyspark version kernels created.) Into this directory put the following into a file named kernel.json:

    {
  "argv": [
    "'${CONDA_PREFIX}'/bin/python",
    "-m",
    "ipykernel_launcher",
    "--profile=pysparkX.Y.Z",
    "-f",
    "{connection_file}"
  ],
  "display_name": "PySpark (Spark X.Y.Z)",
  "language": "python",
  "env": {
    "CAPTURE_STANDARD_OUT": "true",
    "CAPTURE_STANDARD_ERR": "true",
    "SEND_EMPTY_OUTPUT": "false",
    "SPARK_HOME": "~/bin/spark/sparkX.Y.Z"
  }
}

    NOTE: Again, change X.Y.Z to the Spark version number.

  4. Inside the specific conda environment directory (something like <conda-root-dir>/envs/my_env) make sure the following directories exist or create as needed:

    • <conda-root-dir>/envs/my_env/etc/conda/activate.d
    • <conda-root-dir>/envs/my_env/etc/conda/deactivate.d
    • <conda-root-dir>/envs/my_env/share/jupyter/kernels

  5. Into the .../activate.d directory, place a shell script containing: export PATH=$PATH:$HOME/bin/spark/sparkX.Y.Z/bin

    Into the .../deactivate.d directory, place a shell script containing: export PATH=$CONDA_PREFIX/bin:$CONDA_PATH_BACKUP

    From the command line, create a symlink: ln -s <conda-root-dir>/kernels/pysparkX.Y.Z <conda-root-dir>/envs/my_env/share/jupyter/kernels/pysparkX.Y.Z

    NOTE: Once again, change X.Y.Z to the Spark version number.

That's it. When in the activated conda environment, an interactive Spark-enabled command line python session can be started simply by issuing the pyspark command from the command line. An interactive Spark-enabled ipython or jupyter console session can be started with the command ipython --profile=pysparkX.Y.Z or jupyter console --kernel pysparkX.Y.Z, respectively. For a Spark-enabled notebook session, start jupyter notebook normally and then choose the Spark kernel, which should be labeled 'PySpark (Spark X.Y.Z)', when creating a new notebook.

The easy way:

  1. Do step 1 above.
  2. Download this shell script (setup_spark.sh).
  3. Activate your conda environment.

  4. From the command line of this activated environment, execute:

    . \<path-to-script\>/setup_spark.sh X.Y.Z,

    where X.Y.Z is the Spark version number. (But you already knew that, right?)
